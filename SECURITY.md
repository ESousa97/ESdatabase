# Security Policy

## Supported Versions

This project is maintained on a best-effort basis. Security fixes are applied to the latest version.

## Reporting a Vulnerability

Please **do not** open a public issue for security vulnerabilities.

Instead, contact the maintainers privately with:

- Description of the issue
- Steps to reproduce
- Impact assessment (if known)

We will acknowledge receipt and work on a fix.

## Dependency Security

- CI runs `npm audit` (high severity).
- Dependabot is enabled for npm and GitHub Actions updates.
